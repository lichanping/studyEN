## 四、CORS 修复记录

### 4.1 问题描述

原代码中 `fetchCompletedTrainingRecords` 和 `fetchActualReviewList` 直接请求 `https://apiv2.lxll.com`，导致 CORS 错误：

```
Access to fetch at 'https://apiv2.lxll.com/customer/training/orders...' 
from origin 'http://localhost:8888' has been blocked by CORS policy
```

### 4.2 修复方案

**修改文件**: `anti-forgetting.html`

1. `fetchCompletedTrainingRecords` 改为通过 `/.netlify/functions/schedule-board?mode=completed` 代理
2. `fetchActualReviewList` 改为通过 `/.netlify/functions/schedule-board` 代理（传入 mode=anti-forgetting-list）

**修改文件**: `netlify/functions/schedule-board.mjs`

1. 新增 `anti-forgetting-list` mode 支持
2. 接收 `studentName`, `startDate`, `endDate` 参数
3. 调用 `https://apiv2.lxll.com/customer/training/anti-forgetting/list` API

### 4.3 修复验证

| 测试项 | 方法 | 结果 | 状态 |
|-------|------|------|------|
| Netlify function 可访问 | curl 测试 | 返回 502（token 无效，代理正常） | ✅ 通过 |
| CORS 头正确设置 | corsHeaders() 返回 | Access-Control-Allow-Origin: * | ✅ 通过 |
| OPTIONS 预检处理 | 返回 204 | 正确处理预检请求 | ✅ 通过 |

## 六、已知问题和注意事项

### 6.1 本地测试环境要求

1. **需要运行本地服务器**：`python3 -m http.server 8888` 或使用其他 HTTP 服务器
2. **需要有效 token**：在页面中设置有效的 `x-token-c` 和 `x-user-id`
3. **Netlify function 需要正确配置**：确保 `.netlify/functions/schedule-board.mjs` 存在

### 6.2 依赖条件

1. localStorage 中需要存储以下键：
   - `x-token-c`：认证 token
   - `x-user-id`：用户 ID
   - `teacherData_liTeacher_users`：教师学员数据
   - `custom-students-v1`：自定义学员列表
   - `schedule-config-override-v1`：排课配置
   - `schedule-extra-entries-v1`：临时加课记录

### 6.3 API 限制

1. 上游 API `apiv2.lxll.com` 需要有效的认证信息
2. 复习计划只计算从今天开始的记录，已过期的复习日期会被跳过
3. 实际复习列表查询范围为从今天起 30 天内
