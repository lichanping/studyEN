# 验收指南：抗遗忘复习计划核对功能

## 概述

本功能用于核对每个学生实际应排的抗遗忘复习计划，与已排班的实际复习任务进行对比，找出漏排和多排的情况。

---

## 一、功能入口

### 1.1 访问页面
- 打开 `anti-forgetting.html`（抗遗忘复习页）

### 1.2 找到核对按钮
- 在"复习词数"面板中找到 **"📋 核对复习计划"** 按钮
- 位置：复习词数输入框下方，与"抗遗忘课堂反馈"、"新版反馈"并列

---

## 二、使用步骤

### 步骤 1：选择学员
1. 点击"学员"下拉框
2. 选择需要核对的学员姓名
3. 学员来源包括：
   - teacherData 硬编码数据
   - localStorage 自定义学生
   - 排课配置学生
   - 临时加课学生
   - classStatistics 键中的学生名

### 步骤 2：执行核对
1. 点击 **"📋 核对复习计划"** 按钮
2. 按钮状态变为 "⏳ 核对中..."
3. 系统自动执行以下操作：
   - 获取该学生所有已完成的训练记录
   - 根据规则计算应排复习计划
   - 获取实际已排的复习列表
   - 对比差异并展示结果

### 步骤 3：查看结果
核对结果面板会自动显示在按钮下方，包含：

#### 3.1 摘要信息
- ✅ 正常数量（绿色）
- ❌ 漏排数量（红色）
- ⚠️ 多排数量（黄色）

#### 3.2 差异详情表格
- **漏排表格**：复习日期、应排来源、复习节点、状态
- **多排表格**：复习日期、来源、复习节点、状态

#### 3.3 完全正确提示
- 如果无漏排和多排，显示 "🎉 完全正确，所有复习任务均已安排！"

### 步骤 4：导出CSV（可选）
1. 如果有漏排或多排，点击 **"📥 导出CSV"**
2. 文件命名格式：`复习计划核对结果_YYYY-MM-DD.csv`
3. CSV 内容包含：
   - 复习日期
   - 应排来源（正课日期）
   - 复习节点（+1, +2, +3...）
   - 状态（漏排/多排）

---

## 三、核心规则

### 3.1 复习节点规则
正课次日为第 1 天，复习节点为：
```
第 1 天、第 2 天、第 3 天、第 5 天、第 7 天、第 9 天、第 12 天、第 14 天、第 17 天、第 21 天
```

### 3.2 计算示例
假设学生于 **2026-04-22** 上正课：

| 复习节点 | 应排日期 | 
|---------|---------|
| +1 | 2026-04-23 |
| +2 | 2026-04-24 |
| +3 | 2026-04-25 |
| +5 | 2026-04-27 |
| +7 | 2026-04-29 |
| +9 | 2026-05-01 |
| +12 | 2026-05-04 |
| +14 | 2026-05-06 |
| +17 | 2026-05-09 |
| +21 | 2026-05-13 |

### 3.3 核对逻辑
1. **正常**：应排日期有对应复习任务，且来源匹配
2. **漏排**：应排日期无对应复习任务
3. **多排**：实际有复习任务，但无对应正课来源

## 五、API 接口说明

### 5.1 获取已完成训练记录
```
POST https://apiv2.lxll.com/customer/training/orders
参数：
- pageNumber: 页码（从1开始）
- pageSize: 每页数量（最大100）
- status: COMPLETED

请求头：
- x-token-c: {token}
- x-user-id: {userId}
- x-ua: ct=2&v=5.0.96

请求体：
{
  "token": "{token}",
  "userId": "{userId}",
  "xUa": "ct=2&v=5.0.96"
}
```

### 5.2 获取实际复习列表
```
GET https://apiv2.lxll.com/customer/training/anti-forgetting/list
参数：
- studentName: 学员姓名
- startDate: 开始日期（YYYY-MM-DD）
- endDate: 结束日期（YYYY-MM-DD）
- status: ALL

请求头：
- x-token-c: {token}
- x-user-id: {userId}
- x-ua: ct=2&v=5.0.96
```

## 六、常见问题

### Q1：核对结果为空？
**可能原因**：
- 学员无已完成训练记录
- token 无效或未配置

**解决方法**：
1. 点击 "设置登录信息" 重新配置
2. 确认学员有正课记录

### Q2：CSV 文件中文乱码？
**解决方法**：
- 使用 Excel 打开时选择 UTF-8 编码
- 或使用支持 BOM 的文本编辑器

### Q3：应排日期不准确？
**可能原因**：
- 训练记录日期字段名称不一致
- 时区问题

**解决方法**：
- 检查浏览器控制台日志
- 确认 API 返回的日期格式

## 七、技术实现要点

### 7.1 核心函数
| 函数名 | 功能 |
|-------|------|
| `fetchCompletedTrainingRecords()` | 获取已完成训练记录（分页） |
| `calculateReviewSchedule()` | 根据规则计算应排复习计划 |
| `fetchActualReviewList()` | 获取实际复习列表 |
| `compareSchedules()` | 对比应排与实际差异 |
| `renderScheduleCheckResult()` | 渲染核对结果 |

### 7.2 数据流
```
用户点击核对按钮
    ↓
fetchCompletedTrainingRecords(token, userId)
    ↓ 返回 trainingRecords[]
calculateReviewSchedule(trainingRecords, studentName)
    ↓ 返回 expectedSchedule Map<reviewDate, items[]>
fetchActualReviewList(token, userId, startDate, endDate)
    ↓ 返回 actualList[]
compareSchedules(expectedSchedule, actualList)
    ↓ 返回 diff {normal, missing, extra}
renderScheduleCheckResult(diff, studentName, checkDate)
    ↓ 渲染到 DOM
```

### 7.3 关键代码位置
- 文件：`anti-forgetting.html`
- 函数行号：
  - `fetchCompletedTrainingRecords`: 第 518 行
  - `calculateReviewSchedule`: 第 559 行
