# PRD：抗遗忘页昵称映射与今日无复习名单

> 日期：2026-05-08
> 状态：待实现
> 优先级：P0
> 页面：anti-forgetting.html

---

## 1. 背景

当前 `anti-forgetting.html` 已支持：

- 按学员 + 复习日期核对抗遗忘复习计划
- 展示正常 / 漏排 / 多排结果
- 导出 CSV

本次新增两个需求：

1. 修复昵称学员 `硕硕` 无法正确核对的问题
2. 在抗遗忘页增加 `今日无复习` 按钮，输出指定复习日期当天没有抗遗忘复习的学员名单，供人工发送微信文案

---

## 2. 问题定义

### 2.1 P0 Bug：昵称未映射

现有共享映射规则：

- `硕硕 -> 俞新硕`

问题在于 `anti-forgetting.html` 当前未使用共享的 `student-name-alias.js`，导致：

- 页面学员下拉中显示 `硕硕`
- 实际 API 返回的学生名可能是 `俞新硕`
- 复习计划核对时按字符串直接比较，结果会错判为空或漏排

### 2.2 新需求：今日无复习名单

参考 `index.html` 已有 `今日无复习` 按钮的交互习惯，在 `anti-forgetting.html` 中增加一个同名按钮。

点击后，系统需要基于页面当前的 `复习时间`，找出：

- 当前学员列表范围内
- 在该日期当天
- 没有任何抗遗忘复习记录

的学员名单，并输出在页面上供人工复制。

---

## 3. 目标

1. `硕硕` 在抗遗忘页中应按统一映射规则参与核对
2. 抗遗忘页新增 `今日无复习` 按钮
3. 点击按钮后，页面展示指定日期“无抗遗忘复习”的学员名单
4. 保持现有核对功能、CSV 导出、Hint、结果展示不回归

---

## 4. 功能需求

### 4.1 昵称映射

#### 4.1.1 单一事实来源

使用共享文件：

- `student-name-alias.js`

统一调用：

- `StudentNameAlias.normalizeStudentName(name)`

#### 4.1.2 映射应用点

至少应用在以下场景：

1. 选中学员后，核对应排复习计划时的学员名匹配
2. 获取实际抗遗忘列表时的学员名过滤
3. 统计“今日无复习”名单时的学员去重与匹配

#### 4.1.3 展示规则

- 页面下拉仍保留用户习惯名，例如 `硕硕`
- 内部比较时使用标准名，例如 `俞新硕`
- 名单输出时优先展示页面中的学员显示名

### 4.2 今日无复习按钮

#### 4.2.1 入口

在 `anti-forgetting.html` 顶部学员/复习时间区域增加按钮：

- 文案：`今日无复习`

布局参考 `index.html` 的同名按钮，放在较靠上的操作区域。

#### 4.2.2 点击行为

点击后：

1. 读取页面当前 `复习时间`
2. 取日期部分作为核查日期，例如 `2026-05-08`
3. 获取当前页面可选学员列表
4. 逐个学员复用已实现的抗遗忘查询能力，检查当天是否存在实际抗遗忘复习记录
5. 汇总“当天无抗遗忘复习”的学员名单
6. 在页面展示结果

#### 4.2.3 输出内容

页面上新增结果区，至少包含：

- 查询日期
- 无抗遗忘复习学员数量
- 学员名单

展示格式要求：

- 若名单非空：逐行或顿号分隔展示，便于人工复制
- 若名单为空：展示“当天所有学员均有抗遗忘复习”
- 若查询失败：展示错误提示

## 5. 非功能要求

### 5.1 兼容性

- 不改变现有学员下拉逻辑
- 不改变现有核对按钮行为
- 不改变现有 CSV 结构，除非本次需求直接涉及

### 5.2 性能

- 采用现有前端查询方式即可
- 当前学生规模较小，允许逐个学员查询
- 若后续学生规模明显增大，再考虑合并查询优化

### 5.3 可验收性

必须支持本地页面直接验收：

- `http://localhost:8888/anti-forgetting.html`
