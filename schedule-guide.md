# 排课页面使用说明（`schedule.html`）

本文用于说明如何使用 `schedule.html` 查看每天排课，以及如何配置每个学生的 schedule。

## 1. 这个页面能做什么

- 按日期查看“当天排课”
- 自动显示当天总课时
- 一屏查看周一到周日的排课分布
- 自动统计本周总课时

> 页面是“数据驱动”的：你只需要改配置，不需要改渲染代码。

---

## 2. 如何打开使用

1. 在项目里打开 `schedule.html`
2. 在浏览器中访问该文件
3. 在页面顶部使用日期选择器切换日期
4. 查看：
   - 当天课程列表
   - 当天总课时
   - 一周总览和本周总课时

---

## 3. 配置入口在哪里

在 `schedule.html` 底部，找到这个 JSON 配置块：

```html
<script id="schedule-config" type="application/json">
{
  "entries": [
    ...
  ]
}
</script>
```

你只改 `entries` 数组中的内容即可。

---

## 4. 每条排课字段说明

每条记录是一个对象：

```json
{
  "student": "陈怡睿",
  "course": "单词",
  "durationMinutes": 30,
  "days": ["周一", "周二"],
  "period": "晚上",
  "time": ""
}
```

字段含义：

- `student`: 学生姓名（字符串）
- `course`: 课程类型，例如 `单词` / `阅读`
- `durationMinutes`: 时长（分钟），如 `30`、`60`
- `days`: 上课星期数组，只能填 `周一` 到 `周日`
- `period`: 时段，建议填 `上午` / `中午` / `下午` / `晚上`
- `time`: 具体时间（`HH:mm`），例如 `20:00`；如果只确定时段可留空 `""`

---

## 5. 如何配置“每个学生”的 schedule

### 场景 A：每天同一类型课程

例如“陈怡睿：每天半小时单词（晚上）”

```json
{
  "student": "陈怡睿",
  "course": "单词",
  "durationMinutes": 30,
  "days": ["周一", "周二", "周三", "周四", "周五", "周六", "周日"],
  "period": "晚上",
  "time": ""
}
```

### 场景 B：同一学生有多种课程

例如“硕硕：周三单词 + 周六阅读”，需要拆成两条：

```json
{
  "student": "硕硕",
  "course": "单词",
  "durationMinutes": 30,
  "days": ["周三"],
  "period": "晚上",
  "time": ""
},
{
  "student": "硕硕",
  "course": "阅读",
  "durationMinutes": 60,
  "days": ["周六"],
  "period": "晚上",
  "time": "20:00"
}
```

### 场景 C：固定具体时间

例如“胡贝妮：周六日 上午10点 单词课”

```json
{
  "student": "胡贝妮",
  "course": "单词",
  "durationMinutes": 60,
  "days": ["周六", "周日"],
  "period": "上午",
  "time": "10:00"
}
```

## 7. 新增/修改学生操作步骤

1. 打开 `schedule.html`，定位到 `schedule-config` 的 `entries`
2. 按字段格式新增一条（或多条）记录
3. 注意 JSON 语法：
   - 键和字符串都要双引号 `"` 包裹
   - 条目之间用逗号分隔
4. 保存文件并刷新浏览器
5. 用日期选择器检查对应日期是否展示正确

---

## 8. 常见问题

- `days` 写错（如写成 `星期一`）会导致不显示，必须是 `周一`~`周日`
- `durationMinutes` 必须是数字（例如 `60`），不要写成字符串 `"60"`
- `time` 如果有值，建议使用 `HH:mm` 格式（如 `13:30`）
- 如果同一天同一学生有两节课，使用两条记录即可

---

## 9. 推荐维护规范

- 一位学生一周同规则可以合并成一条
- 同一学生不同课程/不同时段，拆成多条
- 每次修改后，用一周中的关键日期快速验一遍（如周三、周六、周日）
